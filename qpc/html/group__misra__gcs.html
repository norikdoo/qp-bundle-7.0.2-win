Guideline Enforcement Plan (GEP)Deviations in QP/C Source Code

As recommended in [MISRA3-Comp], this section presents the MISRA guideline compliance summary, which lists each MISRA-C:2012 rule and indicates the compliance status and how the rule has been checked.

The compliance summary (also referred to as MISRA Compliance Matrix in [MISRA3]) is presented in the tabular form, whereas the meaning of the table columns is as follows:

Directive/Rule

lists the MISRA-C:2012 directive or rule number followed by the rule classification in parentheses:

  • (M) Mandatory — guidelines for wich violation is never permitted
  • (R) Required — guidelines which can only be violated when supported by a deviation defining a set of clear restrictions, requirements and precautions
  • (A) Advisory — recommendations to be followed as far as reasonably practical. Violations are identified but are not required to be supported by deviation.

PCLP

lists whether a given MISRA guideline is checked by PC-lint Plus (see also Configuring PC-lint Plus). This column might contain the following:

  • Yes — the guideline is checked by PC-lint Plus
  • No — the guideline is NOT checked by PC-lint Plus

QP/C

lists the compliance status of the QP/C source code. This column might contain the following:

  • M — means that the QP/C source code complies with the MISRA guideline by Manual inspection
  • A — means that the QP/C source code complies with the MISRA guideline by Automatic inspection (via PC-lint Plus)
  • PQP<num> — denotes a deviation from the MISRA guideline in the QP/C source code with the QP/C Deviation Permit indicated by <num>. NOTE: you can click on the deviation to see the permit.

App

lists the deviations of the Application code imposed by the QP/C framework. This column might contain the following:

  • M — means that the QP/C framewrok does not impose non-compliance of the Application code with the MISRA guideline, but verification of the compliance requires a Manual inspection
  • A — means that the QP/C framewrok does not impose non-compliance of the Application code with the MISRA guideline and the compliance can be verified by Automatic inspection (via PC-lint Plus)
  • PQA<num> — denotes that the QP/C framewrok does impose a deviation from the MISRA guideline in the Application code, but the deviation is granted a Application Deviation Permit indicated by <num>. NOTE: you can click on the deviation to see the permit.

Headline

contains the MISRA guideline headline, as published in [MISRA3].
